调研日期：2026-05-19  
目标平台：macOS、Windows、Linux

## 推荐开发顺序

1. 完成截图捕获引擎的 MVP：区域、窗口、全屏、延时、剪贴板和保存。
2. 接入基础编辑器：裁剪、箭头、矩形、文本、高亮、模糊、撤销重做。
3. 加捕获后浮层和本地历史：复制、保存、拖放、重新打开、最近记录。
4. 加 OCR 与隐私能力：本地文字识别、二维码、邮箱/手机号/密钥快速打码。
5. 加分享与集成：自定义命名、S3/WebDAV/FTP/脚本上传、链接过期和密码。
6. 加录屏和 GIF：先支持区域录制和基本裁剪，再扩展系统音频、摄像头和按键展示。
7. 做平台加固：macOS 权限、Windows 捕获 API、Linux X11/Wayland portal、HiDPI、多显示器和安装更新。

## MVP 建议

MVP 不追求成为 ShareX 式的全能工具，也不先做重云端。第一版应做成一个速度快、稳定、可解释的本地优先工具：

- 一键截图，支持区域、窗口、全屏和延时。
- 选区时有放大镜、尺寸提示、键盘微调和 Esc 退出。
- 截图后出现轻量浮层，允许复制、保存、打开编辑器、拖放到其他应用。
- 编辑器支持常用标注、模糊/像素化、裁剪、撤销重做。
- 历史库保存最近 30 天或用户配置的容量，不强制登录。
- 所有敏感能力默认本地处理，云上传和 AI 必须显式开启。

## 关键架构判断

跨平台截图工具的最大难点不是编辑器，而是捕获权限和桌面环境差异。推荐采用“共享产品层 + 平台适配层”的架构：

- 共享层：捕获请求模型、编辑器图层模型、历史库、工作流规则、上传器接口、设置与快捷键抽象。
- macOS 适配层：屏幕录制权限、窗口枚举、ScreenCaptureKit 或等价原生 API、系统音频策略。
- Windows 适配层：Windows.Graphics.Capture、Print Screen 快捷键、窗口选择、HDR 和缩放处理。
- Linux 适配层：X11 原生捕获、Wayland 下的 xdg-desktop-portal、PipeWire 录屏、全局快捷键降级方案。

如果产品目标是专业用户和开发者，优先级应是：速度、稳定、快捷键、隐私控制、可配置工作流。视觉美化和云协作重要，但不应牺牲本地基本链路。
