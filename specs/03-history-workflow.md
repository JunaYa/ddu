# Spec 03: 历史与工作流

## 目标

把截图工具从一次性捕获工具升级为可复用工作台。用户可以找回最近截图、搜索内容、继续编辑、批量处理，并配置捕获后的自动动作。

## 范围

包含：

- 捕获后浮层。
- 本地历史库。
- 元数据、标签、收藏、搜索。
- 自动命名和保存策略。
- 捕获后动作链和工作流规则。

不包含：

- 云端同步和团队协作，见 [Spec 05](./05-sharing-export-integrations.md)。
- OCR 具体实现，见 [Spec 04](./04-ocr-privacy-ai.md)。

## 用户故事

- 作为开发者，我可以截完图后立即复制、保存、拖到 GitHub issue。
- 作为技术写作者，我可以找回昨天截的某个设置页。
- 作为 QA，我可以把所有带同一标签的截图批量导出。
- 作为高级用户，我可以设置“区域截图后自动复制并保存到指定目录”。
- 作为隐私敏感用户，我可以配置历史保留时间并一键清空。

## 功能需求

### HIS-001 捕获后浮层

捕获完成后显示轻量浮层，默认停留 8 秒，可配置。

浮层必须支持：

- 缩略图预览。
- 复制图片。
- 保存到默认位置。
- 打开编辑器。
- 拖放文件到其他应用。
- 上传或分享。
- 固定到屏幕。
- 删除。
- 恢复最近关闭的浮层。

浮层不应阻塞用户继续操作。多次连续截图时，应以堆叠或队列方式展示。

### HIS-002 历史库

历史库保存：

- 原始截图。
- 编辑后导出图。
- 编辑项目文件。
- 录屏文件。
- OCR 文本。
- 上传记录。
- 捕获来源和应用信息。

默认保留 30 天或 2 GB，以先到为准。用户可配置为 7 天、30 天、90 天、永久或不保存历史。

### HIS-003 元数据

每个历史项记录：

- 捕获时间。
- 捕获模式。
- 应用名、窗口标题、显示器。
- 图片尺寸、文件大小、颜色空间。
- 本地路径。
- 标签、收藏状态、备注。
- OCR 状态和隐私扫描状态。
- 上传目的地、URL、过期时间。

### HIS-004 搜索和筛选

支持：

- 按时间、应用、模式、标签、文件类型筛选。
- 按文件名和窗口标题搜索。
- OCR 完成后按图片内文字搜索。
- 按上传状态和收藏筛选。

搜索必须本地可用。没有 OCR 索引时，搜索结果应显示“文本索引尚未生成”状态。

### HIS-005 自动命名

命名模板支持变量：

- `{date}`、`{time}`、`{datetime}`。
- `{app}`、`{window}`。
- `{mode}`。
- `{counter}`。
- `{width}`、`{height}`。
- `{tag}`。

示例：`{date}/{app}-{mode}-{counter}.png`

命名冲突时自动追加计数器，不覆盖现有文件，除非用户明确选择覆盖。

### HIS-006 工作流动作

内置动作：

- 复制图片到剪贴板。
- 复制文件到剪贴板。
- 保存到默认目录。
- 另存为。
- 打开编辑器。
- 固定到屏幕。
- 上传到默认目的地。
- 复制 URL。
- 运行脚本。
- 打开外部应用。
- 删除本地临时文件。

动作链必须支持排序、启用/禁用和失败处理。

### HIS-007 工作流规则

规则条件：

- 捕获模式。
- 应用名或窗口标题。
- 是否包含 OCR 文本。
- 是否匹配敏感信息。
- 用户按住的修饰键。
- 当前网络状态。

规则示例：

- 区域截图后自动复制并保存。
- 全屏截图后打开编辑器。
- 包含敏感信息时禁止自动上传。
- 按住 Option/Alt 截图时跳过历史保存。

### HIS-008 固定截图

- 用户可把截图固定为置顶无边框窗口。
- 支持调整透明度、大小、位置。
- 支持锁定模式，让鼠标点击穿透到下方应用。
- 固定窗口不应默认进入系统截图结果，避免递归。

## 数据模型

```ts
interface HistoryItem {
  id: string;
  type: "image" | "recording" | "gif" | "document";
  captureId: string;
  sourcePath: string;
  renderedPath?: string;
  projectPath?: string;
  thumbnailPath: string;
  metadata: HistoryMetadata;
  tags: string[];
  favorite: boolean;
  createdAt: string;
  expiresAt?: string;
}

interface WorkflowRule {
  id: string;
  name: string;
  enabled: boolean;
  conditions: WorkflowCondition[];
  actions: WorkflowAction[];
  stopOnFailure: boolean;
}
```

## 验收标准

- 捕获后浮层出现时间小于 200 ms。
- 连续 10 次截图不会丢历史记录或阻塞 UI。
- 用户能按应用名、标签和 OCR 文本找到截图。
- 工作流失败时有明确错误，并且不会丢失原始截图。
- 禁用历史后，应用不写入长期存储，只保留完成当前动作所需临时文件。

## 测试建议

- 单元测试：命名模板、规则匹配、保留策略、动作链失败。
- 集成测试：捕获后复制/保存/编辑/上传流程。
- 压力测试：10,000 条历史记录搜索、缩略图加载、批量删除。
- 隐私测试：禁用历史、清空历史、上传失败、敏感信息拦截。
