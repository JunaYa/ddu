# Spec 04: OCR、隐私与 AI

## 目标

提供可信的本地文字识别和隐私处理能力，让用户可以从截图中复制文本、搜索历史、识别二维码，并在分享前自动发现和隐藏敏感信息。AI 能力必须作为显式可选增强。

## 范围

包含：

- 本地 OCR。
- 二维码识别。
- OCR 文本复制、搜索索引。
- 敏感信息检测和一键打码。
- 可选 AI 处理的权限、边界和输出。

不包含：

- 标注工具的基础渲染，见 [Spec 02](./02-editor-annotation.md)。
- 分享链接治理，见 [Spec 05](./05-sharing-export-integrations.md)。

## 用户故事

- 作为开发者，我可以从错误截图中复制不可选中的错误文本。
- 作为客服，我可以在上传前一键隐藏手机号、邮箱和订单号。
- 作为研究人员，我可以通过 OCR 搜索历史截图。
- 作为跨语言用户，我可以把截图里的文字翻译成另一种语言。
- 作为隐私敏感用户，我可以完全关闭云端 AI。

## 功能需求

### OCR-001 本地 OCR

- 默认在本地运行。
- 支持至少英文和简体中文，后续支持更多语言包。
- 用户可对整张图或选区执行 OCR。
- OCR 结果可复制为纯文本。
- OCR 结果可保存到历史索引。
- OCR 失败时给出原因：语言包缺失、图像过小、权限问题、引擎错误。

### OCR-002 OCR 交互

入口：

- 捕获后浮层：提取文本。
- 编辑器工具栏：OCR 选区。
- 历史库：对选中项生成文本索引。
- 快捷键：直接框选屏幕并复制文本。

结果 UI：

- 显示识别文本。
- 支持复制全部。
- 支持逐行复制。
- 支持保留换行或合并换行。
- 支持打开原图并高亮文本区域。

### OCR-003 二维码识别

- 对整图或选区识别二维码。
- 识别结果显示类型：URL、文本、Wi-Fi、联系人等。
- URL 默认只复制，不自动打开。
- 对可疑 URL 给出安全提示。

### OCR-004 敏感信息检测

内置检测类型：

- 邮箱。
- 手机号和电话号码。
- 身份证号或本地化证件号，按地区开关。
- 银行卡号。
- URL query token。
- API key 常见模式。
- JWT。
- IPv4/IPv6。
- 地址和姓名作为增强能力，不进入 MVP 自动打码。

检测结果必须可预览，用户确认后才应用打码，除非用户明确开启自动打码规则。

### OCR-005 智能打码

打码方式：

- 模糊。
- 像素化。
- 实色覆盖。
- 文本擦除。

规则：

- 默认使用不可逆像素化或实色覆盖。
- 导出前再次扫描可见图像，提示仍存在的敏感信息。
- 上传动作触发前，如果检测到高风险敏感信息且未处理，必须拦截或二次确认。

### OCR-006 OCR 搜索索引

历史库索引字段：

- 原始 OCR 文本。
- 标准化文本。
- 语言。
- 识别置信度。
- 文本区域坐标。
- 生成时间和引擎版本。

索引必须本地保存。用户关闭历史或 OCR 索引时，应删除对应文本数据。

### OCR-007 可选 AI 能力

AI 功能必须满足：

- 默认关闭。
- 首次使用解释会发送什么内容。
- 支持仅发送用户选区。
- 支持自动先打码再发送。
- 提供模型/服务商配置。
- 对每次请求记录本地审计日志。

可选 AI 功能：

- 截图摘要。
- 生成 alt 文本。
- 翻译 OCR 文本。
- 从截图提取表格。
- 根据截图生成 bug 复现草稿。
- 智能裁剪和重点区域建议。

## 数据模型

```ts
interface OcrResult {
  id: string;
  historyItemId: string;
  language: string;
  engine: "system" | "tesseract" | "cloud" | "custom";
  text: string;
  blocks: OcrBlock[];
  confidence: number;
  createdAt: string;
}

interface SensitiveFinding {
  id: string;
  type:
    | "email"
    | "phone"
    | "idNumber"
    | "paymentCard"
    | "urlToken"
    | "apiKey"
    | "jwt"
    | "ipAddress";
  textPreview: string;
  bounds: Rect[];
  confidence: number;
  suggestedAction: "pixelate" | "solidFill" | "blur";
}
```

## 隐私要求

- 本地 OCR 不应上传图片或文本。
- 云 AI 请求必须有可见确认和设置开关。
- 历史库中 OCR 文本应跟随历史保留策略删除。
- 用户清空历史时，缩略图、原图、OCR 文本、AI 请求日志和上传缓存都必须清理。
- 自动上传前必须检查当前项的隐私扫描状态。

## 验收标准

- 对清晰英文和中文截图，OCR 可在本地完成并复制文本。
- QR 识别结果不会自动打开外部链接。
- 邮箱、手机号、常见 API key 能被检测并一键打码。
- 关闭 AI 后，所有 AI 入口显示不可用状态，不发送任何请求。
- 上传前敏感信息拦截规则生效。

## 测试建议

- OCR 样例：英文错误栈、中文聊天、表格、低分辨率图片、深色主题代码。
- 隐私样例：邮箱、手机号、银行卡、JWT、GitHub token、URL token。
- 安全测试：离线模式、关闭 AI、清空历史、上传前拦截。
- 回归测试：打码后导出图片不可通过项目文件恢复原文。
